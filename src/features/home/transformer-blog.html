Section 01

Introduction

In 2017, a team at Google published a paper titled "Attention Is All You Need", introducing the Transformer — a neural network architecture that would go on to become the backbone of modern AI. GPT, BERT, T5, LLaMA — they all trace their lineage back to this single paper.

The key insight was radical: instead of processing sequences step-by-step using recurrent networks, why not let every token attend to every other token simultaneously? This simple idea — self-attention — unlocked parallelism, improved long-range reasoning, and enabled the scaling of models to billions of parameters.

"We propose a new simple network architecture, the Transformer, based solely on attention mechanisms, dispensing with recurrence and convolutions entirely."

— Vaswani et al., Attention Is All You Need, 2017
Section 02

Why Transformers?

To understand why transformers matter, we need to appreciate what came before them.

Model How it works Problem
RNN Processes tokens one at a time, left to right, passing a hidden state forward Sequential → slow. Struggles with long-range dependencies due to vanishing gradients.
LSTM Same as RNN but with gating mechanisms to better preserve long-range context Still sequential. Still slow for long sequences. Complex to train.
Transformer All tokens processed in parallel. Every token attends to all others. Large memory usage for very long sequences, but otherwise highly scalable.

The core advantage: Transformers process an entire sequence at once, not token-by-token. During training, this means massive GPU parallelism. During inference, it means each token can directly "see" any other token, regardless of distance.

Architecture Overview

The Transformer is built from two main stacks: an Encoder and a Decoder. Each stack is composed of identical layers stacked on top of each other (typically 6).

👁️

Each token computes how much to attend to every other token.

⚡

⚖️

Layer Norm

Normalizes activations after each sub-layer for stable training.

🔀

Cross-Attention

Decoder attends to encoder output to align input and output.

Section 04

Step 1 — Token Embedding

Before a neural network can process text, we need to convert words (or subword tokens) into vectors of numbers. An embedding layer is simply a lookup table: each token ID maps to a learned vector of size embed_dim.

In the original paper, the model dimension is d_model = 512, meaning each token becomes a 512-dimensional vector. These embeddings are learned during training — the model discovers what geometric relationships between vectors best help it solve the task.

1
Tokenize

Text is split into tokens (words, subwords, or characters) and converted to integer IDs using a vocabulary.

2
Lookup

Each integer ID indexes into an embedding matrix of shape [vocab_size × d_model], retrieving its row.

3
Output

A sequence of n tokens becomes a matrix of shape [n × d_model], ready for the transformer layers.

Embedding Class Python · PyTorch 
class Embedding(nn.Module):
    def __init__(self, vocab_size, embed_dim):
        """
        vocab_size : total number of unique tokens in vocabulary
        embed_dim  : dimension of each token's vector (d_model)
        """
        super(Embedding, self).__init__()
        # nn.Embedding is a learnable lookup table
        self.embed = nn.Embedding(vocab_size, embed_dim)

    def forward(self, x):
        """
        x   : LongTensor of shape [batch_size, seq_length]
               containing token IDs
        out : FloatTensor of shape [batch_size, seq_length, embed_dim]
        """
        out = self.embed(x)
        return out
Section 05

Step 2 — Positional Encoding

Self-attention is permutation-invariant — it doesn't care about the order of tokens by default. If you shuffle all words in a sentence, the attention mechanism would produce the same output (modulo the shuffled positions). This is clearly a problem.

The solution is positional encoding: we add a unique signal to each token's embedding that encodes its position in the sequence. The original Transformer uses sinusoidal functions, which have a useful property — they generalize to sequence lengths unseen during training.

Positional Encoding Formulas
PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i + 1) = cos(pos / 10000^(2i/d_model))
Where pos is the position in the sequence and i is the dimension index. Even dimensions use sin, odd dimensions use cos.

Why does this work? Each position gets a unique pattern of sine and cosine waves at different frequencies. The model can learn to use these signals to determine relative distances between tokens — e.g., "this word is 3 positions after that word."

Positional Encoding Python · PyTorch 
class PositionalEncoding(nn.Module):
    def __init__(self, d_model, max_seq_length):
        super(PositionalEncoding, self).__init__()

        # Create a matrix of shape [max_seq_length, d_model]
        pe = torch.zeros(max_seq_length, d_model)

        # position: column vector [0, 1, 2, ..., max_seq_length-1]
        position = torch.arange(0, max_seq_length, dtype=torch.float).unsqueeze(1)

        # div_term: the 1/10000^(2i/d_model) frequency scaling factors
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * -(math.log(10000.0) / d_model)
        )

        # Apply sin to even dimensions, cos to odd dimensions
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)

        # Store as non-trainable buffer; add batch dim: [1, max_seq_length, d_model]
        self.register_buffer('pe', pe.unsqueeze(0))

    def forward(self, x):
        # Add positional encoding to embedding.
        # x shape: [batch_size, seq_length, d_model]
        return x + self.pe[:, :x.size(1)]

Key insight: We add positional encodings to embeddings — they occupy the same vector space. The model learns to disentangle meaning (from the embedding) and position (from the encoding) jointly. The encodings are fixed, not learned, so they're always consistent.

Section 06

Step 3 — Multi-Head Attention

This is the heart of the Transformer. The attention mechanism allows each token to gather information from every other token in the sequence, weighted by relevance. The "multi-head" variant runs several attention operations in parallel, allowing the model to attend to different types of relationships simultaneously.

Scaled Dot-Product Attention

Each token is projected into three roles: a Query (what am I looking for?), a Key (what do I offer?), and a Value (what information do I carry?).

Attention Formula
Attention(Q, K, V) = softmax( QKᵀ / √d_k ) · V
Q = queries, K = keys, V = values, d_k = head dimension. The √d_k scaling prevents the dot products from getting too large, which would push softmax into regions with near-zero gradients.
1
Project Q, K, V

Linear layers W_q, W_k, W_v project the input into Query, Key, and Value matrices.

2
Compute Attention Scores

Dot product of Q and Kᵀ gives raw scores. Divide by √d_k to scale. Apply causal mask if in decoder.

3
Softmax → Weights

Softmax converts scores to probabilities (attention weights) that sum to 1 over all positions.

4
Weighted Sum of Values

Multiply weights by V to produce the output: a weighted blend of all values, biased toward the most relevant ones.

5
Multi-Head & Combine

Run h attention heads in parallel on lower-dimensional projections, then concatenate and project back with W_o.

Multi-Head Attention Python · PyTorch 
class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super(MultiHeadAttention, self).__init__()
        assert d_model % num_heads == 0, "d_model must be divisible by num_heads"

        self.d_model    = d_model
        self.num_heads  = num_heads
        self.d_k        = d_model // num_heads  # each head operates on d_k dims

        # Input projections for Q, K, V and the output projection
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def scaled_dot_product_attention(self, Q, K, V, mask=None):
        # QKᵀ / √d_k  →  shape: [batch, heads, seq, seq]
        attn_scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)

        # Mask: fill -1e9 (≈ -∞) before softmax so those positions get ~0 weight
        if mask is not None:
            attn_scores = attn_scores.masked_fill(mask == 0, -1e9)

        attn_probs = torch.softmax(attn_scores, dim=-1)
        return torch.matmul(attn_probs, V)

    def split_heads(self, x):
        # [batch, seq, d_model] → [batch, heads, seq, d_k]
        batch_size, seq_length, d_model = x.size()
        return x.view(batch_size, seq_length, self.num_heads, self.d_k).transpose(1, 2)

    def combine_heads(self, x):
        # [batch, heads, seq, d_k] → [batch, seq, d_model]
        batch_size, _, seq_length, d_k = x.size()
        return x.transpose(1, 2).contiguous().view(batch_size, seq_length, self.d_model)

    def forward(self, Q, K, V, mask=None):
        # Project and split across heads
        Q = self.split_heads(self.W_q(Q))
        K = self.split_heads(self.W_k(K))
        V = self.split_heads(self.W_v(V))

        # Compute attention for all heads in parallel
        attn_output = self.scaled_dot_product_attention(Q, K, V, mask)

        # Combine heads and apply output projection
        return self.W_o(self.combine_heads(attn_output))

Why multiple heads? Different heads can learn to specialize — one might attend to syntactic relationships, another to coreference, another to semantic similarity. With 8 heads and d_model = 512, each head operates on 64 dimensions, but collectively they capture a rich, multi-faceted view of the sequence.

Section 07

Step 4 — Position-Wise Feed-Forward Network

After attention aggregates information across positions, a feed-forward network (FFN) is applied to each position independently. This is a simple two-layer MLP with a ReLU activation in the middle.

Think of attention as the "communication" step (tokens talk to each other), and the FFN as the "computation" step (each token individually processes what it learned). The inner dimension d_ff is typically 4× the model dimension (e.g., 2048 when d_model = 512), giving the model extra capacity.

FFN Formula
FFN(x) = max(0, x·W₁ + b₁)·W₂ + b₂
Two linear transformations with a ReLU activation between them. Applied identically to each position.
Position-Wise Feed-Forward Network Python · PyTorch 
class PositionWiseFeedForward(nn.Module):
    def __init__(self, d_model, d_ff):
        """
        d_model : model dimension (e.g. 512)
        d_ff    : inner dimension, typically 4 × d_model (e.g. 2048)
        """
        super(PositionWiseFeedForward, self).__init__()
        self.fc1  = nn.Linear(d_model, d_ff)   # expand
        self.fc2  = nn.Linear(d_ff, d_model)   # contract
        self.relu = nn.ReLU()

    def forward(self, x):
        # x: [batch, seq, d_model]
        # Expand → activate → contract
        return self.fc2(self.relu(self.fc1(x)))
Section 08

Step 5 — Encoder Layer

A single encoder layer combines Multi-Head Self-Attention and the Feed-Forward Network, each wrapped with a residual connection and layer normalization. This "Add & Norm" pattern is critical for training stability in deep networks.

The encoder receives the full source sequence and produces a rich contextual representation for each token. These representations are then passed to the decoder via cross-attention.

1
Self-Attention

Each token attends to all tokens in the input (including itself). The padding mask ensures PAD tokens don't influence results.

2
Add & Norm

The attention output is added to the original input (residual), then layer-normalized. Residuals enable gradient flow through deep stacks.

3
Feed-Forward

The normalized output goes through the position-wise FFN for further non-linear transformation.

4
Add & Norm (again)

Another residual connection + layer norm after the FFN. Output shape matches input: [batch, seq, d_model].

Encoder Layer Python · PyTorch 
class EncoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout):
        super(EncoderLayer, self).__init__()
        self.self_attn    = MultiHeadAttention(d_model, num_heads)
        self.feed_forward = PositionWiseFeedForward(d_model, d_ff)
        self.norm1        = nn.LayerNorm(d_model)
        self.norm2        = nn.LayerNorm(d_model)
        self.dropout      = nn.Dropout(dropout)

    def forward(self, x, mask):
        # --- Sub-layer 1: Self-Attention ---
        attn_output = self.self_attn(x, x, x, mask)          # Q=K=V=x (self-attention)
        x = self.norm1(x + self.dropout(attn_output))         # residual + norm

        # --- Sub-layer 2: Feed-Forward ---
        ff_output = self.feed_forward(x)
        x = self.norm2(x + self.dropout(ff_output))           # residual + norm

        return x  # shape: [batch, seq, d_model]
Section 09

Step 6 — Decoder Layer

The decoder is where generation happens. It takes the target sequence so far and uses it — combined with the encoder's output — to predict the next token. It has three sub-layers instead of two:

🎭

Masked Self-Attention

Like encoder self-attention, but with a causal mask — each token can only attend to itself and earlier tokens. Prevents "cheating" by looking at future tokens.

🔗

Cross-Attention

Queries come from the decoder, but Keys and Values come from the encoder output. This is how the decoder "looks at" the source sequence.

⚡

Feed-Forward

Same as the encoder — independent position-wise MLP for added model capacity.

The causal mask is an upper-triangular matrix of −∞ values. When added to attention scores before softmax, positions in the future get zero weight. This enforces autoregressive generation: the model generates tokens one at a time, only conditioning on what it has already produced.

Decoder Layer Python · PyTorch 
class DecoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout):
        super(DecoderLayer, self).__init__()
        self.self_attn    = MultiHeadAttention(d_model, num_heads)  # masked
        self.cross_attn   = MultiHeadAttention(d_model, num_heads)  # cross
        self.feed_forward = PositionWiseFeedForward(d_model, d_ff)
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.norm3 = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, enc_output, src_mask, tgt_mask):
        # --- Sub-layer 1: Masked Self-Attention (causal) ---
        attn_output = self.self_attn(x, x, x, tgt_mask)
        x = self.norm1(x + self.dropout(attn_output))

        # --- Sub-layer 2: Cross-Attention ---
        # Q = decoder state, K = V = encoder output
        attn_output = self.cross_attn(x, enc_output, enc_output, src_mask)
        x = self.norm2(x + self.dropout(attn_output))

        # --- Sub-layer 3: Feed-Forward ---
        ff_output = self.feed_forward(x)
        x = self.norm3(x + self.dropout(ff_output))

        return x
Section 10

Step 7 — Putting It All Together

The full Transformer class assembles all the components: source and target embeddings, positional encodings, a stack of encoder layers, a stack of decoder layers, and a final linear + softmax head to produce output probabilities.

Mask Generation

Before the forward pass, two masks are created:

①
Source mask (padding mask)

Marks which source positions are real tokens (1) vs padding (0), so padding tokens are ignored during attention.

②
Target mask (causal + padding)

Combines a padding mask with the causal "no-peak" mask to prevent attending to future target tokens.

Full Transformer Python · PyTorch 
class Transformer(nn.Module):
    def __init__(self, src_vocab_size, tgt_vocab_size,
                 d_model, num_heads, num_layers,
                 d_ff, max_seq_length, dropout):
        super(Transformer, self).__init__()

        # ── Embeddings (separate for source and target) ──────────────────────
        self.encoder_embedding = Embedding(src_vocab_size, d_model)
        self.decoder_embedding = Embedding(tgt_vocab_size, d_model)
        self.positional_encoding = PositionalEncoding(d_model, max_seq_length)

        # ── Stacks of encoder and decoder layers ─────────────────────────────
        self.encoder_layers = nn.ModuleList(
            [EncoderLayer(d_model, num_heads, d_ff, dropout) for _ in range(num_layers)]
        )
        self.decoder_layers = nn.ModuleList(
            [DecoderLayer(d_model, num_heads, d_ff, dropout) for _ in range(num_layers)]
        )

        # ── Output projection: d_model → tgt_vocab_size ──────────────────────
        self.fc = nn.Linear(d_model, tgt_vocab_size)
        self.dropout = nn.Dropout(dropout)

    def generate_mask(self, src, tgt):
        # Padding mask: shape [batch, 1, 1, src_len]
        src_mask = (src != 0).unsqueeze(1).unsqueeze(2)

        # Target padding mask: shape [batch, 1, tgt_len, 1]
        tgt_mask = (tgt != 0).unsqueeze(1).unsqueeze(3)

        # Causal (no-peak) mask: lower-triangular matrix of 1s
        seq_length = tgt.size(1)
        nopeak_mask = (
            1 - torch.triu(torch.ones(1, seq_length, seq_length), diagonal=1)
        ).bool()

        # Combine padding mask with causal mask
        tgt_mask = tgt_mask & nopeak_mask
        return src_mask, tgt_mask

    def forward(self, src, tgt):
        src_mask, tgt_mask = self.generate_mask(src, tgt)

        # ── Encode source sequence ────────────────────────────────────────────
        src_embedded = self.dropout(
            self.positional_encoding(self.encoder_embedding(src))
        )
        enc_output = src_embedded
        for enc_layer in self.encoder_layers:
            enc_output = enc_layer(enc_output, src_mask)

        # ── Decode target sequence ────────────────────────────────────────────
        tgt_embedded = self.dropout(
            self.positional_encoding(self.decoder_embedding(tgt))
        )
        dec_output = tgt_embedded
        for dec_layer in self.decoder_layers:
            dec_output = dec_layer(dec_output, enc_output, src_mask, tgt_mask)

        # ── Project to vocabulary ─────────────────────────────────────────────
        output = self.fc(dec_output)
        # shape: [batch, tgt_seq_length, tgt_vocab_size]
        return output

What happens during a forward pass? Source tokens → embeddings + positional encoding → 6 encoder layers (each doing self-attention + FFN) → encoder output. Target tokens → embeddings + positional encoding → 6 decoder layers (each doing masked self-attention + cross-attention + FFN) → linear projection → logits over target vocabulary → softmax → next token probabilities.
